# Component 3: Physical Control & Spatial Safety

## 1. Overview
The "Body" module is responsible for executing the Mission Plan generated by the Brain. It translates GPS waypoints and "Pointing" commands into motor-level instructions for the **Unitree G1 Humanoid**, ensuring every movement is safe, stable, and collision-free.

## 2. Autonomous Navigation & Path Planning
Instead of hard-coded paths, the robot uses a dynamic navigation stack:
- **GPS-to-Local Mapping:** Converts Global Map coordinates (from the UI) into a local Cartesian grid ($x, y, z$).
- **Obstacle Avoidance:** Uses the G1's onboard **LiDAR and Depth Cameras** to build a real-time occupancy map.
- **Dynamic Re-planning:** If a human or object blocks the path, the robot recalculates a trajectory in real-time without stopping the tour, fulfilling the "beyond tool calls" requirement.

## 3. Gestural Interaction (The "Point" Function)
To make the guide immersive, the robot must physically interact with its surroundings:
- **Kinematic Mapping:** Takes coordinates from the Vision Module (Component 2) and calculates the necessary joint angles for the G1's arms using **Inverse Kinematics (IK)**.
- **Safe Motion Limits:** Software "virtual fences" prevent the arms from moving too fast or colliding with the robot’s own torso.
- **Visual Confirmation:** The robot aligns its head (camera) and arm simultaneously toward the landmark to create a natural "look and point" behavior.

## 4. Safety & Collision Guardrails
Safety is the top priority for deployment on the shared Unitree fleet:
- **The "Safety Bubble":** A 0.5-meter proximity sensor trigger. If any object enters this bubble, the robot immediately transitions to a `STANCE_LOCK` (stable standing) state.
- **Nebius Simulation:** Before executing a complex arm gesture or a new walking gait, the movement is validated in a **supported simulation stack** (MuJoCo or Isaac Gym) on Nebius Cloud to ensure it won't cause the robot to fall.
- **Emergency Stop (E-Stop):** A software-level listener that can be triggered from the Web UI to freeze all motor activity instantly.

## 5. Deployment & Execution Loop
1. **Receive Waypoint:** Get the next (x, y) target.
2. **Compute Path:** Generate a collision-free route.
3. **Locomotion:** Trigger the G1 walking gait.
4. **Task Execution:** Upon arrival, trigger the "Point and Talk" sequence.
5. **Feedback:** Send "Task Complete" status back to the Backend.

## 6. Technical Requirements
- **Hardware:** Unitree G1 Humanoid, LiDAR, IMU.
- **Compute:** Nebius AI Cloud for RL/Policy training and motion simulation.
- **Safety Metric:** Zero unplanned physical contacts during the 3-minute demo.
